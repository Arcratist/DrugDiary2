ARCHITECTURE IMPLEMENTED
================================================================================

Fragment (UI)
    ↓ creates via ViewModelFactory
ViewModel (Lifecycle-aware)
    ↓ uses
ServiceLocator (dependency injection)
    ↓ provides
Repository Interfaces (domain contracts)
    ↓ delegates to
Repository Implementations (data layer)
    ↓ uses
Storage/Network (actual I/O)

Benefits:
✅ Separation of concerns
✅ Lifecycle management (survives config changes)
✅ Easy testability (mock repositories)
✅ Reactive UI (LiveData updates)
✅ Error handling
✅ Thread safety
